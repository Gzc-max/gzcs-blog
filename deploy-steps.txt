# 博客部署步骤 - 复制粘贴即可

## ========== 第一步：推送到 GitHub ==========

### 1. 初始化 Git（如果还没有）
git init

### 2. 添加所有文件
git add .

### 3. 提交
git commit -m "初始化博客项目，准备部署"

### 4. 在 GitHub 创建新仓库
打开浏览器访问: https://github.com/new
- Repository name: 输入仓库名（如: my-blog 或 personal-blog）
- Description: 个人博客项目
- Public 或 Private: 选择 Public（公开）
- 不要勾选 "Add a README file"
- 点击 "Create repository"

### 5. 关联远程仓库（创建仓库后，GitHub 会显示这些命令）
# 将下面的 YOUR_USERNAME 和 YOUR_REPO 替换为你的用户名和仓库名
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git

### 6. 推送代码
git branch -M main
git push -u origin main

## ========== 第二步：部署到 Vercel ==========

### 方式 A：网页部署（推荐，最简单）

1. 打开浏览器访问: https://vercel.com

2. 点击右上角 "Sign Up" 注册
   - 选择 "Continue with GitHub"（用 GitHub 账号登录）
   - 点击 "Authorize Vercel" 授权

3. 登录后点击 "Add New..." → "Project"

4. 找到你的博客仓库，点击 "Import"

5. 确认配置（通常自动填写，无需修改）：
   - Project Name: 你的项目名
   - Framework Preset: Vite（自动识别）
   - Build Command: npm run build（自动填写）
   - Output Directory: dist（自动填写）

6. 点击 "Deploy" 按钮

7. 等待 1-2 分钟，部署完成！

8. 点击显示的网址访问你的博客
   例如: https://your-blog.vercel.app

### 方式 B：命令行部署（可选）

# 1. 安装 Vercel CLI
npm install -g vercel

# 2. 登录
vercel login

# 3. 部署
vercel

# 4. 生产环境部署
vercel --prod

## ========== 第三步：后续更新 ==========

### 每次修改代码后，只需执行：
git add .
git commit -m "更新内容描述"
git push

# Vercel 会自动检测并重新部署！

## ========== 常见问题 ==========

Q: 如果推送时要求输入密码？
A: GitHub 现在需要使用 Personal Access Token
   访问: https://github.com/settings/tokens
   生成 token，用 token 作为密码

Q: 如果提示 remote origin already exists？
A: 执行: git remote remove origin
   然后重新执行关联命令

Q: 部署失败怎么办？
A: 检查 Vercel 的部署日志，通常会显示具体错误
   或者联系我帮你解决
